# AGENTS.md — how to work on the Iliamo menu site

This repository is the **Iliamo Kitchen & Bar** menu — a static website published at
https://www.iliamo.bar via **Cloudflare Pages**. There is no backend and no framework.

The people who ask you to make changes are **not programmers**. They will usually write
to you in **Russian**, describing menu changes in plain words (prices, dishes, photos,
links). Your job is to turn those requests into small, safe edits and publish them.

Keep the menu text in the language the owner uses (Russian). Prices are in Georgian
lari and are written like `"8 ₾"`.

---

## The one rule that matters

**All menu content lives in `data/menu.json`. It is the single source of truth.**
Edit that file, then run the build. Never hand-edit the menu anywhere else — the block
inside `index.html` is generated by the build and will be overwritten.

Every change follows the same three steps:

1. Edit `data/menu.json`.
2. Run the build (this is mandatory — the live site will not update without it):
   ```
   python3 build.py
   ```
   It must finish with no `BUILD FAILED` message. If it fails, read the message —
   it tells you exactly what is wrong (bad JSON, a price with no number, a photo file
   that does not exist) — fix it and run again.
3. Commit and push. **Pushing publishes the change** — Cloudflare Pages redeploys
   `iliamo.bar` automatically within about a minute.
   ```
   git add -A
   git commit -m "menu: <short description in English>"
   git push
   ```

Before committing you may preview locally: `python3 server.py` serves the site at
http://localhost:8000/ .

## `data/menu.json` structure

```
brand        — name, subtitle, logo, siteUrl, and links[] (Instagram, maps, ...)
categories[] — the buttons across the top: { id, title, label, theme, icon }
sections[]   — one per category id: { id, title, label, theme, note, groups[] }
  groups[]   — { title, note, items[] }   (title "" means an unnamed group)
    items[]  — { name, price?, desc?, icon, image?, lead? }
```

An item:
```json
{ "name": "Латте", "price": "10 ₾", "desc": "", "icon": "coffee" }
```

Fields:
- `name` — required.
- `price` — optional string like `"10 ₾"`. Omit it when the price is stated in the
  section `note` instead (e.g. tea flavours, sauces).
- `desc` — optional short description / ingredients.
- `icon` — the little glyph; reuse the icon the neighbouring items in that section use
  (e.g. `beer`, `cocktail`, `shot`, `wine`, `soft`, `coffee`, `tea`, `food`, `salad`,
  `soup`, `sauce`, `lemonade`).
- `image` — optional photo, e.g. `"assets/menu/fries.webp"` (see Photos below).
- `lead: true` — marks the one item that becomes the big highlighted card in a section
  listed in `leadSections`. Keep at most one `lead` per section.

`theme` is either `"drink"` or `"food"`. A `section` id must always match a `category`
id — every section needs a category button, and vice versa.

## Common requests → what to change

| The owner says (RU) | What you do |
| --- | --- |
| «подними Латте до 11 ₾» | find the item `Латте`, set `"price": "11 ₾"` |
| «добавь в Закуски начос за 12 ₾» | add an item to the `snacks` section's group |
| «убери Bloody Mary» | delete that item object |
| «переименуй … / поправь описание» | edit `name` / `desc` |
| «добавь новую категорию Десерты» | add a `categories[]` entry **and** a matching `sections[]` entry with the same `id` |
| «поменяй фото шавермы» | replace/add the file in `assets/menu/`, set `image` (see below) |
| «поменяй ссылку на инстаграм» | edit `brand.links[]` |

When adding a category, pick a short latin `id` (e.g. `desserts`), reuse an existing
`theme` and `icon`, and add the section right after in `sections[]`.

## Photos

- Put image files in `assets/menu/`. **WebP** is preferred (smallest); `.jpg`/`.png`
  also work. If the owner gives you a large `.jpg`/`.png`, convert it to `.webp` when
  you can (e.g. `cwebp input.jpg -o assets/menu/name.webp`), otherwise reference the
  file as given.
- Reference it from the item: `"image": "assets/menu/name.webp"`.
- The build checks that every referenced photo exists; a typo will fail the build with
  a clear message.

## What NOT to touch (advanced)

For ordinary menu work you only ever edit `data/menu.json`. The following are code /
layout and should be changed only on an explicit design/layout request:

- `app.js` — site logic. If you edit it, the build copies it to `app.min.js` for you.
- `styles.css` — styling. `index.html` loads the **minified** `styles.min.css` plus a
  small critical-CSS block inlined at the top of `index.html`. So if you change
  `styles.css` you must also regenerate `styles.min.css`
  (e.g. `npx --yes csso-cli styles.css -o styles.min.css`) and update the inline
  critical block in `index.html` if you touched a rule used there. Then run
  `python3 build.py` to bump the cache version.
- `index.html` structure, `_headers`, `server.py`, `build.py` — infrastructure.
- `.pipeline/` — internal notes, ignored by git; leave it alone.

Never commit secrets, tokens, or personal data — this is a **public** repository.

---

## Sanity check before you say "done"

- `python3 build.py` exits cleanly (no `BUILD FAILED`).
- `python3 build.py --check` prints `OK`.
- The change is what the owner asked for, and you pushed it.
